# העלאה לאינטרנט — קישור קבוע לשיתוף 🌐

מדריך זה מסביר איך להעלות את המערכת לאינטרנט ולקבל **קישור ציבורי קבוע** שתוכל לשלוח למשתמשים שתרשום.

איך זה עובד: אתה (מנהל) נכנס למערכת → **משתמשים** → מוסיף משתמש עם שם וסיסמה ורמת הרשאה (מנהל / עורך / צפייה בלבד) → שולח לאדם את הקישור ואת פרטי הכניסה.

---

## אפשרות מומלצת: Render.com (חינמי, בלי כרטיס אשראי, בלי CLI)

### שלב 1 — העלאת הקוד ל-GitHub
1. פִּתחו חשבון ב-[github.com](https://github.com) (חינמי).
2. צרו Repository חדש (למשל `family-events`).
3. העלו את כל קבצי הפרויקט (אפשר דרך הכפתור **"uploading an existing file"** בממשק GitHub, או עם Git).
   > אין צורך להעלות את התיקיות `node_modules` ו-`data` — הן ממילא ב-`.gitignore`.

### שלב 2 — פריסה ב-Render
1. היכנסו ל-[render.com](https://render.com) והתחברו עם חשבון GitHub.
2. **New → Web Service** ובחרו את ה-Repository שהעליתם.
3. Render יזהה את הקובץ `render.yaml` אוטומטית. אם לא, הגדירו ידנית:
   - **Build Command:** `npm install`
   - **Start Command:** `npm start`
   - **Environment:** הוסיפו משתנה `NODE_ENV=production` ו-`ADMIN_PASSWORD=<סיסמה חזקה משלכם>`.
4. לחצו **Create Web Service**. אחרי 1–2 דקות תקבלו קישור קבוע כמו:
   **`https://family-events.onrender.com`** — זה הקישור לשיתוף! 🎉

### שלב 3 — כניסה ראשונה
- שם משתמש: `admin`
- סיסמה: מה שהגדרתם ב-`ADMIN_PASSWORD` (אם לא הגדרתם — `1234`, **שנו אותה מיד** דרך הגדרות → החלפת סיסמה).

## ⚠️ חשוב לגבי שמירת נתונים בתוכנית החינמית

בתוכנית החינמית של Render, שטח הדיסק **זמני** — בכל פריסה מחדש או "הרדמה" של השירות (לאחר חוסר פעילות) הנתונים עלולים להתאפס.

**פתרונות:**
1. **גיבוי ידני (הכי פשוט):** מתוך המערכת → **הגדרות → גיבוי ושחזור → ייצוא גיבוי**. שומר קובץ עם כל הנתונים. אחרי איפוס — **שחזור מקובץ** מחזיר הכול.
2. **דיסק קבוע (התמדה מלאה):** דורש תוכנית בתשלום ב-Render (~7$/חודש). בטלו את ההערה מבלוק ה-`disk` בקובץ `render.yaml` והוסיפו `DATA_DIR=/var/data`.
3. **חלופות עם התמדה חינמית:** Railway.app או Fly.io תומכים ב-Volumes קבועים (קצת יותר מורכבים להגדרה).

## שליחת מיילים אמיתית (אופציונלי)

כדי שהתזכורות יישלחו במייל אמיתי: היכנסו ל**הגדרות → דוא"ל (SMTP)** והזינו פרטי שרת SMTP.
לדוגמה עם Gmail: צרו "סיסמת אפליקציה" בחשבון Google והזינו:
`smtp.gmail.com` · פורט `587` · המשתמש והסיסמה שיצרתם.
