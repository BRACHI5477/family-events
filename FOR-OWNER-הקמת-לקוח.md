> הוא מסביר איך את מקימה מערכת עצמאית לכל לקוח חדש, בכ-20 דקות.

הלקוח מקבל בסוף **רק 3 דברים**: קישור לאתר, שם משתמש וסיסמה.
הוא לא נוגע בקוד, לא רואה GitHub, ולא רואה אף לקוח אחר.

## הכנה חד-פעמית (פעם אחת בלבד)

- ✅ הריפו שלך ב-GitHub — **מומלץ להפוך אותו לפרטי**:
  GitHub → הריפו → **Settings** → למטה **Danger Zone** → **Change visibility** → **Make private**.
  (המערכת שלך והלקוחות ימשיכו לעבוד — הפרטיות לא משפיעה על אתרים שכבר רצים.)

## לכל לקוח חדש — 4 שלבים

### שלב 1 · מסד נתונים (Neon) — 3 דקות
1. [console.neon.tech](https://console.neon.tech) → **New Project**
2. שם הפרויקט: שם הלקוח (למשל `cohen-family`)
3. **Connect** → **Show password** → **Copy** את מחרוזת ה-`DATABASE_URL`

### שלב 2 · שליחת מיילים (Brevo) — אפשר לחלוק חשבון אחד לכל הלקוחות
- אפשר להשתמש **באותו חשבון Brevo** שלך לכל הלקוחות (300 מיילים/יום משותפים).
- אם לקוח שולח הרבה — פתחי לו חשבון Brevo נפרד.
- הפרטים שתצטרכי: `SMTP_USER` (ה-Login) ו-`SMTP_PASS` (ה-key).

### שלב 3 · הפעלת הסקריפט — מדפיס לך את כל משתני הסביבה מוכנים
מתוך תיקיית הפרויקט, הריצי:
```
npm run new-client
```
ועני על השאלות (שם הלקוח, המייל שלו, סיסמה, וה-DATABASE_URL מ-Neon).
הסקריפט **ידפיס לך טבלה מוכנה** של כל משתני הסביבה — פשוט להעתיק ל-Render.

### שלב 4 · פריסה (Render) — 5 דקות
1. [render.com](https://render.com) → **New** → **Web Service**
2. בחרי את הריפו `family-events` (שלך)
3. **Name:** שם הלקוח (זה יקבע את הכתובת: `שם-הלקוח.onrender.com`)
4. **Instance Type:** Free · **Language:** Docker
5. **Environment Variables** → הדביקי את מה שהסקריפט הדפיס
6. **Create Web Service** → אחרי ~4 דקות יש כתובת

**בדיקה:** פתחי `https://שם-הלקוח.onrender.com/api/health` → צריך `"cloud_db":true`.

### שלב 5 · פינג (שהשרת לא יירדם) — 2 דקות
[cron-job.org](https://console.cron-job.org) → **Create Cronjob**:
- URL: `https://שם-הלקוח.onrender.com/api/health`
- **Every 10 minutes** → **SAVE**

## מה מוסרים ללקוח
שלחי לו הודעה כזו:
```
שלום! המערכת שלך מוכנה 🎉
כתובת: https://שם-הלקוח.onrender.com
שם משתמש: <המייל שלו>
סיסמה: <הסיסמה שקבעת>

מומלץ להחליף סיסמה אחרי הכניסה הראשונה: הגדרות → החלפת סיסמה.
```

**זהו.** הלקוח לא מקבל שום דבר אחר. הנתונים שלו במסד הנפרד שלו, ואת לא רואה אותם.

- **עדכון לכל הלקוחות:** דוחפים שינוי ל-GitHub → כל אתרי Render מתעדכנים אוטומטית.
