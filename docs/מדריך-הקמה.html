חינם לחלוטין כ-30 דקות ללא ידע טכני

מה תקבלו

מערכת אינטרנטית עצמאית לחלוטין — עם הנתונים שלכם, המנהל שלכם והכתובת שלכם. אף אחד אחר לא רואה את הנתונים, לא יכול לשנות או למחוק אותם, ושום עדכון אצל מישהו אחר לא ישפיע עליכם.

  • 🗓️ לוח שנה עברי (בגימטריה) ולועזי
  • 👥 בני משפחה, אירועים, וחישוב גיל אוטומטי (בר מצווה = 13)
  • 🔔 תזכורות מייל אוטומטיות — כולל מיקום, הערות ותמונה
  • 📧 12 תבניות מייל מעוצבות (יום הולדת, בר/בת מצווה, חתונה, ברית, יארצייט ועוד)
  • 🏘️ ניהול כמה משפחות במקביל
  • 🔐 משתמשים והרשאות (מנהל / עורך / צפייה בלבד)
  • 💾 גיבוי, ייצוא נתונים, ואיפוס סיסמה במייל

1העתקת הקוד — GitHub

  1. פִּתחו חשבון חינם ב-github.com
  2. היכנסו לכתובת: github.com/BRACHI5477/family-events
  3. לחצו על כפתור Fork (למעלה מימין) ← Create fork

זה יוצר עותק פרטי של הקוד בחשבון שלכם.

2מסד הנתונים — Neon

⚠️ זה השלב הקריטי ביותר. בלעדיו כל הנתונים שלכם יימחקו כל כמה שעות!
  1. היכנסו ל-console.neon.tech/signup ← Continue with GitHub
  2. צרו פרויקט (אפשר להשאיר את שם ברירת המחדל)
  3. במסך הפרויקט לחצו Connect ← Show password ← Copy snippet
  4. שמרו את המחרוזת בפנקס הרשימות. היא נראית כך:

postgresql://neondb_owner:סיסמה@ep-xxxx.aws.neon.tech/neondb?sslmode=require

3שליחת מיילים — Brevo

  1. הירשמו ב-brevo.com/free-account
  2. לכו ל-SMTP & API ← צרו SMTP key ← שמרו אותו מיד (מוצג פעם אחת בלבד!)
  3. באותו מסך רשמו לעצמכם את ה-Login (משהו כמו b1xxxxx@smtp-brevo.com)
  4. לכו ל-Senders ← הוסיפו את המייל שלכם כשולח ← אשרו במייל האימות שיישלח אליכם

4פריסת האתר — Render

  1. היכנסו ל-render.com ← Get Started ← Sign in with GitHub
  2. New ← Web Service ← בחרו את הריפו family-events שהעתקתם
  3. ודאו: Language = Docker · Branch = main · Instance Type = Free
  4. גללו ל-Environment Variables והוסיפו את 10 השורות מהטבלה שלמטה
  5. לחצו Create Web Service. אחרי כ-4 דקות תקבלו כתובת כמו https://השם-שלכם.onrender.com

משתני הסביבה

בכל שורה יש שתי תיבות: שמאלית = השם (באנגלית, בדיוק כפי שכתוב), ימנית = הערך.

⚠️ טעות נפוצה: למלא רק את התיבה הימנית ולהשאיר את השמאלית ריקה. שתי התיבות חייבות להיות מלאות בכל שורה.

5בדיקה שהכל תקין

  1. היכנסו לכתובת שקיבלתם, עם שם המשתמש והסיסמה שהגדרתם
  2. הגדרות ← 🔌 בדיקת חיבור SMTP ← אמור להופיע "החיבור ל-SMTP תקין!"
  3. פִּתחו בדפדפן: https://הכתובת-שלכם.onrender.com/api/health

צריך להופיע "cloud_db":true.

אם מופיע "cloud_db":false — ה-DATABASE_URL לא נקלט, והנתונים שלכם יימחקו! חזרו לשלב 4 ובדקו את השורה הזו.

6שהשרת לא יירדם (מומלץ)

בתוכנית החינמית השרת נרדם אחרי חוסר פעילות, ואז תזכורות עלולות להישלח באיחור.

  1. הירשמו ב-cron-job.org
  2. צרו Cronjob חדש:
    • URL: https://הכתובת-שלכם.onrender.com/api/health
    • Schedule: כל 10 דקות
💡 שימו לב: הפינג אינו שולח מיילים. הוא רק "נוגע" בשרת כדי שיישאר ער.

שאלות נפוצות

האם מישהו אחר יכול לראות או למחוק את הנתונים שלי?

לא. מסד הנתונים (Neon) והשרת (Render) הם שלכם בלבד.

מה אם המערכת המקורית תתעדכן?

לא ישפיע עליכם — אתם על עותק (fork) משלכם.

האתר איטי בכניסה הראשונה?

בתוכנית החינמית השרת "מתעורר" (כ-40 שניות). שלב 6 פותר זאת.

שכחתי את הסיסמה

לחצו "שכחתי סיסמה" במסך הכניסה. יישלח מייל עם קישור לבחירת סיסמה חדשה. כל הנתונים נשמרים — רק הסיסמה משתנה.

נטפרי חוסם את הכתובת

נפוץ עם דומיינים חדשים. שלחו לנטפרי בקשה לפתיחת הכתובת.

איך אני נותן גישה לבני משפחה?

מסך משתמשים ← הוספת משתמש ← בחרו הרשאה ← לחצו 📧 כדי לשלוח לו במייל קישור התחברות עם הפרטים.

איך שומרים גיבוי?

הגדרות ← גיבוי ושחזור ← ייצוא גיבוי. שומר קובץ עם כל הנתונים.
