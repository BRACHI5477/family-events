הרעיון בקצרה

לכל לקוח את מקימה מערכת עצמאית ונפרדת (כ-20 דקות עבודה). הלקוח מקבל בסוף רק 3 דברים: קישור לאתר, שם משתמש וסיסמה. הוא לא נוגע בקוד, לא רואה GitHub, ולא רואה אף לקוח אחר. הנתונים שלו במסד הנפרד שלו — ואת לא רואה אותם.

הכנה חד-פעמית

הפכי את הריפו ב-GitHub לפרטי: הריפו ← Settings ← למטה Danger Zone ← Change visibility ← Make private.
זה מונע מאנשים אחרים לראות או להעתיק את הקוד. אתרים שכבר רצים לא יושפעו.

לכל לקוח חדש — השלבים

1מסד נתונים (Neon) · 3 דק'

  1. console.neon.tech ← New Project
  2. שם הפרויקט = שם הלקוח (למשל cohen-family)
  3. Connect ← Show password ← Copy את מחרוזת ה-DATABASE_URL

2שליחת מיילים (Brevo)

אפשר להשתמש באותו חשבון Brevo שלך לכל הלקוחות (300 מיילים ביום, משותפים). שמרי בהישג יד את ה-SMTP_USER (ה-Login) ואת ה-SMTP_PASS (ה-key).

3הפעלת הסקריפט

מתוך תיקיית הפרויקט, הריצי:

npm run new-client

ענו על השאלות (שם הלקוח, המייל שלו, סיסמה, וה-DATABASE_URL). הסקריפט מדפיס טבלה מוכנה של כל משתני הסביבה — פשוט להעתיק ל-Render.

4פריסה (Render) · 5 דק'

  1. render.com ← New ← Web Service
  2. בחרי את הריפו family-events (שלך)
  3. Name = שם הלקוח (זה קובע את הכתובת: שם-הלקוח.onrender.com)
  4. Instance Type: Free · Language: Docker
  5. Environment Variables ← הדביקי את מה שהסקריפט הדפיס
  6. Create Web Service ← אחרי ~4 דקות יש כתובת
בדיקה: פתחי https://שם-הלקוח.onrender.com/api/health ← צריך להופיע "cloud_db":true. אם false — ה-DATABASE_URL לא נקלט, והנתונים יימחקו!

5פינג — שהשרת לא יירדם · 2 דק'

  1. console.cron-job.org ← Create Cronjob
  2. URL: https://שם-הלקוח.onrender.com/api/health
  3. Every 10 minutes ← SAVE

מה מוסרים ללקוח

שלום! המערכת שלך מוכנה 🎉
כתובת: https://שם-הלקוח.onrender.com
שם משתמש: <המייל שלו>
סיסמה: <הסיסמה שקבעת>
מומלץ להחליף סיסמה אחרי הכניסה: הגדרות ← החלפת סיסמה.

זהו. הלקוח לא מקבל שום דבר אחר.
