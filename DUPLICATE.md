# שכפול המערכת — יצירת מערכות עצמאיות 📑

מדריך זה מסביר איך ליצור **עותקים עצמאיים לגמרי** של המערכת — אחד לכל משפחה/לקוח, שכל אחד מנהל לבד. לכל עותק יש מסד נתונים משלו, מנהל משלו, כתובת משלו ומיתוג משלו — אין ביניהם שום קשר.

> הבחנה חשובה: זה **שונה** מ"כמה משפחות במערכת אחת". כאן כל עותק הוא מערכת נפרדת ומבודדת.

---

## יצירת עותק חדש — בפקודה אחת

מתוך תיקיית המערכת המקורית, הריצו:

```bash
npm run new-instance
```

תישאלו כמה שאלות (שם המערכת, שם המנהל, סיסמה, פורט, תיקיית יעד), ותיווצר תיקייה חדשה עם מערכת עצמאית. בסוף מוצגות הוראות ההפעלה.

### או עם קובץ קונפיג (ללא שאלות)

צרו קובץ `client.json`:

```json
{
  "systemName": "יומן משפחת ישראלי",
  "familyName": "משפחת ישראלי",
  "mode": "single",
  "logo": "🏠",
  "port": 3005,
  "target": "C:/מערכות/ישראלי",
  "owner": { "username": "israeli@gmail.com", "password": "בחרו-סיסמה", "name": "ישראל ישראלי" }
}
```

והריצו:

```bash
node scripts/new-instance.js client.json
```

**שדות:**
- `systemName` — שם המערכת (בכותרת ובמיילים).
- `familyName` — שם המשפחה הראשונה שתיווצר.
- `mode` — `single` (משפחה אחת, המנהל = admin) או `multi` (המנהל = מנהל-על שיכול לפתוח כמה משפחות).
- `port` — פורט הרצה (לכל עותק פורט שונה אם רצים על אותו מחשב).
- `target` — תיקיית היעד (אם מושמט — תיקייה לצד המקור לפי שם המערכת).
- `owner` — פרטי המנהל שייווצר (שם משתמש, סיסמה, שם מלא).

---

## הפעלת העותק

```bash
cd <תיקיית-היעד>
npm install
npm start
```

פתחו את הכתובת שמוצגת (למשל `http://localhost:3005`) והתחברו עם פרטי המנהל שהגדרתם.

מה תקבלו בעותק חדש:
- ✅ מסד נתונים ריק משלו (בלי נתוני דמו, בלי משתמשים אחרים).
- ✅ מנהל משלו לפי הקונפיג — **אין** מנהלת-על גלובלית (ברכי) ואין משתמש `admin`.
- ✅ שם ומיתוג משלו.
- ✅ 12 סוגי אירועים סטנדרטיים מוכנים לשימוש.

---

## הרצת כמה עותקים על אותו מחשב

תנו לכל עותק **פורט שונה** (נשמר ב-`instance.config.json` של אותו עותק), והפעילו כל אחד בחלון נפרד:

```
עותק א' → http://localhost:3005
עותק ב' → http://localhost:3006
```

## העלאת עותק לאינטרנט

כל עותק אפשר להעלות לענן בנפרד לפי [DEPLOY.md](DEPLOY.md) — כל אחד יקבל כתובת ציבורית משלו. הגדירו את פרטי המנהל דרך `instance.config.json` או משתני סביבה (`OWNER_USERNAME`, `OWNER_PASSWORD`, `SYSTEM_NAME`).

---

## איך זה עובד מאחורי הקלעים

קובץ `instance.config.json` בשורש העותק הוא מה שהופך אותו לעצמאי. כשהוא קיים, המערכת (ב-`server/instanceConfig.js`) יוצרת בהפעלה הראשונה מנהל ומשפחה לפי הקונפיג — בלי נתוני הדמו ובלי מנהלת-העל. אם הקובץ חסר, המערכת עולה במצב המקורי (מאסטר) עם ברכי כמנהלת-על.
