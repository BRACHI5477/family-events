# 🏠 איך להקים יומן אירועים משפחתי משלך

מדריך זה מיועד למי שרוצה **מערכת עצמאית לחלוטין** — עם הנתונים שלו, המנהל שלו והכתובת שלו.

> **חשוב:** המערכת שתקים תהיה **נפרדת לגמרי**. אף אחד אחר לא יראה את הנתונים שלך, לא יוכל לשנות או למחוק אותם, ושום עדכון אצל מישהו אחר לא ישפיע עליך.

**עלות: 0 ₪.** הכל בתוכניות חינמיות.
**זמן: כ-30 דקות.**

## שלב 1 — העתקת הקוד (GitHub)

1. פִּתחו חשבון ב-[github.com](https://github.com) (חינם).
2. היכנסו ל: **https://github.com/BRACHI5477/family-events**
3. לחצו על כפתור **"Fork"** (למעלה מימין) → **Create fork**.
   זה יוצר עותק פרטי של הקוד בחשבון שלכם.

---

## שלב 2 — מסד נתונים (Neon) 🔑

**זה השלב הכי חשוב.** בלעדיו הנתונים יימחקו כל כמה שעות.

1. היכנסו ל-[console.neon.tech/signup](https://console.neon.tech/signup) → **Continue with GitHub**.
2. צרו פרויקט (אפשר להשאיר את שם ברירת המחדל).
3. במסך הפרויקט לחצו **"Connect"** → **"Show password"** → **"Copy snippet"**.
4. שמרו את המחרוזת. היא נראית כך:
   ```
   postgresql://neondb_owner:סיסמה@ep-xxxx.aws.neon.tech/neondb?sslmode=require
   ```

---

## שלב 3 — שליחת מיילים (Brevo)

1. הירשמו ב-[brevo.com](https://www.brevo.com/free-account/).
2. **הגדרות → SMTP & API** → צרו **SMTP key** ושמרו אותו (מוצג פעם אחת בלבד!).
3. רשמו לעצמכם מהמסך: **SMTP Server** (`smtp-relay.brevo.com`) ואת ה-**Login** (משהו כמו `b1xxxxx@smtp-brevo.com`).
4. **Senders** → הוסיפו את המייל שלכם כשולח, ואשרו אותו במייל האימות שיישלח אליכם.

---

## שלב 4 — פריסת האתר (Render)

1. היכנסו ל-[render.com](https://render.com) → **Get Started** → **Sign in with GitHub**.
2. **New → Web Service** → בחרו את הריפו `family-events` שהעתקתם.
3. הגדרות:
   - **Language:** `Docker` (מזוהה אוטומטית)
   - **Branch:** `main`
   - **Instance Type:** `Free`
4. גללו ל-**Environment Variables** והוסיפו את השורות הבאות.
   בכל שורה: **תיבה שמאלית = השם** (באנגלית, בדיוק כך), **תיבה ימנית = הערך**.

| השם (Key) | הערך (Value) | מה זה |
|---|---|---|
| `DATABASE_URL` | המחרוזת מ-Neon | 🔑 שומר את הנתונים לנצח |
| `SUPERADMIN_USERNAME` | המייל שלכם | שם המשתמש שלכם לכניסה |
| `SUPERADMIN_PASSWORD` | סיסמה שתבחרו | הסיסמה שלכם לכניסה |
| `SUPERADMIN_NAME` | השם שלכם | יוצג בממשק |
| `SMTP_HOST` | `smtp-relay.brevo.com` | שרת המייל |
| `SMTP_PORT` | `2525` | ⚠️ **חייב 2525** — פורט 587 חסום ב-Render |
| `SMTP_USER` | ה-Login מ-Brevo | משתמש לשליחה |
| `SMTP_PASS` | ה-SMTP key מ-Brevo | סיסמת השליחה |
| `SENDER_EMAIL` | המייל המאומת שלכם | ממי נשלח |
| `SENDER_NAME` | `יומן אירועים משפחתי` | שם השולח |

5. לחצו **Create Web Service**. אחרי ~4 דקות תקבלו כתובת כמו:
   **`https://השם-שלכם.onrender.com`**

---

## שלב 5 — כניסה ראשונה

היכנסו לכתובת שקיבלתם, עם `SUPERADMIN_USERNAME` ו-`SUPERADMIN_PASSWORD` שהגדרתם.

**בדיקה שהכל תקין:**
- **הגדרות → 🔌 בדיקת חיבור SMTP** → אמור להופיע *"החיבור ל-SMTP תקין!"*
- פִּתחו את `https://הכתובת-שלכם.onrender.com/api/health` — צריך להופיע `"cloud_db":true`
  (אם מופיע `false` — ה-`DATABASE_URL` לא נקלט, והנתונים יימחקו!)

## שלב 6 (מומלץ) — שהשרת לא יירדם

בתוכנית החינמית השרת נרדם אחרי חוסר פעילות, ואז תזכורות עלולות להישלח באיחור.

1. הירשמו ב-[cron-job.org](https://cron-job.org) (חינם).
2. צרו Cronjob חדש:
   - **URL:** `https://הכתובת-שלכם.onrender.com/api/health`
   - **Schedule:** כל 10 דקות
3. שמרו. זהו — השרת יישאר ער והתזכורות יישלחו בזמן.

## שאלות נפוצות

**האם מישהו אחר יכול לראות או למחוק את הנתונים שלי?**
לא. המסד (Neon) והשרת (Render) הם שלכם בלבד.

**מה אם המערכת המקורית תתעדכן?**
לא ישפיע עליכם. אתם על עותק (fork) משלכם. אם תרצו לקבל שיפורים — תוכלו לסנכרן ידנית מ-GitHub.

**האתר איטי בכניסה הראשונה?**
בתוכנית החינמית השרת "מתעורר" (~40 שניות). שלב 6 פותר זאת.

**נטפרי חוסם את הכתובת?**
זה נפוץ עם דומיינים חדשים. שלחו לנטפרי בקשה לפתיחת הכתובת.

**איך אני נותן גישה לבני משפחה?**
מסך **משתמשים** → הוספת משתמש → בחרו הרשאה (מנהל / עורך / צפייה בלבד) → שלחו להם את הקישור ואת הפרטים.
